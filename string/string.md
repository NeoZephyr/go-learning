# 字符串

## string

在底层，一个 string 类型的值是由一系列相对应的 Unicode 代码点的 UTF-8 编码值来表达的。既可以被拆分为一个包含多个字符的序列，也可以被拆分为一个包含多个字节的序列。前者可以由一个以 rune 为元素类型的切片来表示，而后者则可以由一个以 byte 为元素类型的切片代表

string 包含一个指向字节数组头部的指针值，一个 string 会在底层与它的所有副本共用同一个字节数组。由于这里的字节数组永远不会被改变，所以 string 类型的值是不可变的。如果想获得一个不一样的字符串，那么就只能基于原字符串进行裁剪、拼接等操作，从而生成一个新的字符串

## strings.Builder

与 string 值存储内容的方式一样，Builder 值中有一个 byte 切片用于承载内容，它们都通过一个 unsafe.Pointer 指针指向底层字节数组

虽然，对于字节切片本身来说，它包含的任何元素值都可以被修改，但是 Builder 值并不允许这样做，其中的内容只能够被拼接或者完全重置。

由于已使用的 Builder 值不能再被复制，所以不会出现多个 Builder 值中的字节切片共用一个底层字节数组的情况。这样就避免了多个同源的 Builder 值在拼接内容时可能产生的冲突问题
